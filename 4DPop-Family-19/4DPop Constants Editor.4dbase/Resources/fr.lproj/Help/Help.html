4DPop
Constants Editor

4DPop Constant Editor permet de créer et éditer des constantes personnalisées.

Installation

Placer le dossier "4DPop Constant Editor.4dbase" ou un alias de ce dossier dans le dossier "Components" côté de la structure de votre base de donnée.

Si vous utilisez le composant "4DPop", l'outil "Constants" apparaît automatiquement dans la palette d'outils.

Vous ouvrez l'éditeur en cliquant sur l'outil.

Vous pouvez aussi lancer manuellement l'éditeur en exécutant la méthode "Constant_Editor"via le menu "Exécution > Méthode..."


Utilisation

Les constantes créées et éditées sont celles présentes dans le plugin "User Constants.bundle" (cf. 4D_v11_Mise_A_Jour.pdf).
Si le plugin n'est pas installé, il le sera automatiquement si vous créez des constantes.

La liste de gauche de l'éditeur présente les constantes personnalisées regroupées par thème. La sélection d'un item permet d'éditer ses propriétés dans la partie droite de la fenêtre.

Les boutons de la barre d'outils permettent respectivement de gauche à droite : de créer un thème, d'ajouter une constante à un thème ou bien de supprimer l'élément sélectionné.
Les item du menu Fichier permettent d'ajouter un thème, une constante ou d'importer des constantes depuis un fichier.

Note : La localisation des constantes personnalisées a été modifiée en v11. Si vous convertissez une base utilisant des constantes personnalisées situées dans la structure reportez vous à la documentation de 4D v11 SQL (4D_v11_Mise_A_Jour.pdf) et à celle du composant 4DPop Migration qui vous permet de transférer vos constantes afin qu'elles puissent être chargées au démarrage.

À la fermeture de l'éditeur, celui-ci vous demande si vous voulez enregistrer les modifications effectuées et vous propose dans l'affirmative de redémarrer la base afin que les modifications soient prises en compte.

Client-Serveur

Il n'est pour le moment pas possible d'éditer les constantes de la bse en cours en mode client-serveur.
